# OtoWeave プロトタイプテスト手順書

所要時間の目安: 45〜60分（シナリオ1〜6）
記録先: `フィードバックシート.md`（各シナリオの記入欄に対応）

> ストップウォッチ（スマホで可）をご用意ください。
> 「かかった時間」はおおよそで構いません。

---

## 準備: セットアップがまだの場合

1. zip で受け取った場合は、展開する前に zip を右クリック →
   「プロパティ」→「許可する」にチェック → OK
   （Windows のブロックを外す操作です。項目が無ければ不要です）
2. フォルダごとローカルディスクへコピー
   - コピー先は `C:\OtoWeave` のような**浅い場所**がおすすめです
     （深いフォルダだとパスが長すぎてエラーになることがあります）
3. 「setup.bat」をダブルクリックしてセットアップ
   - 最初に空き容量のチェックがあります。足りない場合は
     表示された量まで空きを増やしてから再実行してください
   - 配布は**どの端末にも標準版1本**でかまいません。セットアップが
     端末のメモリ・空き容量を自動判定して、AI要約を「使える設定」に
     するか「使わない設定（モデルを削除して約2.5GB節約）」にするかを
     決めます（判定結果は画面と `setup_report.txt` に表示されます）
   - Python 3.12 が無い端末でも、同梱インストーラーで自動的に
     インストールされます
4. 「実行がブロックされました」等と表示されて進めない場合は、
   学校の管理者（情報担当の先生など）に相談してください
5. 最後に「すべてOKです」と出れば準備完了です
   （結果は `setup_report.txt` にも保存されます。フィードバックと
   一緒に回収させてください）

## シナリオ0: 起動確認（1分）

1. 「OtoWeaveを起動.bat」をダブルクリック
2. **計測**: ダブルクリックからウィンドウが操作できるまでの秒数
3. 画面の文字が読みやすいか（フォント・サイズ・色）を第一印象でメモ

## シナリオ1: 録音と リアルタイム文字起こし（10分）

1. 左端の 🔴（録音）→「マイク」を選び「3秒テスト」で音量を確認
2. 「録音を開始」→「日本語」を選択
3. 2〜3分、普通の声で話す（教科書の音読などでOK）
   - **確認**: 話してから文字が出るまでの体感遅延（何秒くらい？）
   - **確認**: 「未処理 N 区間・遅れあり」の表示が出たか
4. 話しながら「★重要」「？」「！」ボタンを1回ずつ押す
5. 上へスクロールして「↓ 最新へ戻る」ボタンが出るか確認 → 押して復帰
6. 一時停止 → 10秒待つ → 再開（時間表示が止まっていたか確認）
7. 停止 → ノートが一覧に現れるまでの時間を計測
8. **精度チェック**: 文字起こしを読み、間違っている箇所に印を
   つけるつもりで数える（だいたい10文中いくつ間違いがあるか）

## シナリオ2: 聞き直しと読み上げ（5分）

1. シナリオ1のノートを開く
2. ▶ で再生 → Ⅱ で一時停止できるか
3. 文字起こしの**青い時刻（00:05 など）をクリック** → その位置から
   再生されるか
4. 「🔊 読み上げ」を押す → 合成音声で本文が読み上げられるか
   - **確認**: 声の聞き取りやすさ（5段階でメモ）
   - もう一度押して止まるか

## シナリオ3: AI要約とチャット（15分・待ち時間あり）

> **前提条件（要約）**: AI要約が使えるかどうかは、セットアップ時の
> **自動判定**で決まります（目安: メモリ16GBクラスの端末で有効）。
> **メモリ8GB以下の端末（GIGA端末など）や Lite版では、「要約を作成」
> ボタンは表示されず、「AIようやくは、この端末ではじゅんび中です」と
> 案内が出ます。これは自動判定の正常な動作で、故障ではありません。**
> その場合は手順1〜5を飛ばし、**案内が表示されること**を確認して
> 手順6（チャット）だけ行ってください。チャットはどの構成でも使えます。
> （どちらの判定だったかは `setup_report.txt` で確認できます）

1. 右パネルでテンプレート「授業の要点」を選び「要約を作成」
2. **計測**: 完了までの分数（この間、他の操作の反応が重くないか）
3. 途中で一度「キャンセル」を試す → すぐ止まるか → もう一度実行
4. できた要約を読み、**元の話に無い内容が混ざっていないか**確認
5. 「🔊 読み上げ」で要約を聞く
6. チャット欄で内容について1つ質問 → 回答時間と妥当性をメモ

## シナリオ4: 音声ファイルの取り込み（10分）

1. スマホ等で録音した音声ファイル（または任意のMP3/WAV）を用意
2. 📥（取り込み）から選択、またはウィンドウへドラッグ＆ドロップ
3. 「日本語」を選び、完了まで放置
   - **計測**: 音声の長さと処理時間（例: 10分の音声→何分？）
4. 取り込み中に別のノートを開いてみる → 表示が乱れないか

## シナリオ5: 表示設定（5分）

1. ⚙（設定）でフォント・文字サイズ・行間・文字はば・配色（ダーク）を
   ひととおり変更
2. **確認**: それぞれ本文にすぐ反映されるか／読みやすくなったか
3. 「最新の文字起こしを追う」をOFF → 録音して追従が止まるか（余裕があれば）

## シナリオ6: 保存の安心感（5分）

1. ノートを1つ削除 → 確認ダイアログの説明がわかりやすいか
2. `ドキュメント\OtoWeave\_trash` にフォルダが移動しているか確認
3. アプリを×で閉じて再起動 → ノートが全部残っているか

## シナリオ7: 機内モード確認 — 外部に送信していないことの確認（10分）

> このアプリは、AIも含めてすべて端末の中だけで動きます。
> インターネットを切っても全機能が動くことを実際に確認するシナリオです。
> くわしい説明は `docs\データの取り扱いと確認方法.md` にあります。

1. Windowsの画面右下のネットワークアイコンから**機内モードをON**
   （またはWi-Fiをオフ。有線LANの場合はケーブルを抜く）
2. ブラウザで適当なページを開き、「接続されていません」と
   表示されることを確認（回線が本当に切れている証拠）
3. OtoWeaveで次を順に試し、**すべて動くこと**を確認
   - 1〜2分の録音 → 文字起こしが出る
   - 「🔊 読み上げ」で本文が読み上げられる
   - チャット欄でAIに1つ質問 → 回答が返る
   - （AI要約が有効な端末なら）「要約を作成」ができる
4. 機内モードをOFFに戻す

**あわせて（余裕があれば）**: 配布フォルダの `verify_offline.ps1` を
右クリック →「PowerShellで実行」してから上記の操作を行うと、
外部への通信が0件であることが `offline_report.txt` に記録されます
（標準で10分間監視して自動終了します。フィードバックと一緒に
回収させてください）。

## シナリオ8（任意・開発者/検証担当者向け）: 話者分離とAI要約(9B)の処理時間実測

> 話者分離はまだプロトタイプで、OtoWeaveアプリ本体には組み込まれて
> いません。このシナリオは一般のテスト参加者は実施不要です。低スペック機
> （GIGA端末クラス）やメモリ16GB機での実測データを集めるための、
> 開発者・検証担当者向けの追加計測です。

### 8-1. 話者分離の処理時間（低スペック機での実測）

参考値（i5-1145G7・4スレッドで実測）: RTF=0.174、71分音声の処理に
約12.4分、ピークメモリ約580MB。低スペック機（Celeron級のGIGA端末など）
では2〜4倍遅くなる可能性があり、実機での確認が必要です。

1. `scripts\diarization_prototype.py` と `models\diarization\`
   （セグメンテーション+話者埋め込みの2モデル、計約46MB）は
   配布フォルダに同梱済みです（`build_distribution.ps1` が標準版・
   Lite版とも同梱します）。測定機にコピーした配布フォルダにこの2つが
   あることを確認してください
2. 71分程度の音声ファイル（授業録音など）を用意し、次を実行

   ```powershell
   .\.venv\Scripts\python.exe scripts\diarization_prototype.py `
     --audio "C:\path\to\71min_audio.wav" `
     --output_dir runs\diarization_test `
     --num_speakers 2 `
     --num_threads 4 `
     --max_seconds 4260
   ```

   （`--max_seconds` は71分=4260秒。音声の長さに合わせて調整してください。
   話者数が分かっている場合は `--num_speakers` で指定すると安定します）
3. **計測**: コンソールに出る `RTF=`（実時間比）と
   `Diarization done in ... s`（処理秒数）
4. 完了後 `runs\diarization_test\run_meta.json` の
   `real_time_factor` と `process_seconds` を記録
5. タスクマネージャー等でこの間のピークメモリ（作業セット）も記録

### 8-2. AI要約(9B)の所要時間（メモリ16GB機での実測）

対象: メモリ16GBクラスの端末（9Bモデルが自動的に優先利用される構成）。

1. `distribution\download_9b_model.ps1` を実行し9Bモデルを取得
   （搭載メモリを確認し、15GB以下の端末では警告のうえ確認を求めます）
2. `setup_test_pc.ps1` を（再）実行し、9Bが「残しました（AI要約は
   上位棚9Bを優先利用）」と判定されることを `setup_report.txt` で確認
3. シナリオ3の手順1〜2と同様に「要約を作成」を実行し、完了までの
   時間を計測
4. 4Bモデルのみの端末（またはLite版）との体感差（速度・要約の質）をメモ

## 記録してほしい環境情報

- PCの機種名 / メモリ容量（設定→システム→バージョン情報）
- 使ったマイク（内蔵 / 外付け）
- テスト場所の環境音（静か / ざわざわ / 授業中 など）

## もし問題が起きたら

- 画面が固まった・エラーが出た → **そのままの画面を写真に撮る**
- 録音が保存されていない場合も、`ドキュメント\OtoWeave` の
  該当フォルダをそのまま残しておいてください（原因調査に使います）
