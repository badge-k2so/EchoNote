# OtoWeave Mac (Apple Silicon) テスト手順書

対象機種の目安: MacBook（M1/M2/M3など Apple Silicon搭載）、メモリ8GB以上。
所要時間の目安: セットアップ20〜30分＋テスト45〜60分。

> これは **検証中のMac版** です。Windows版に比べて実機での確認がまだ
> 少ないため、うまくいかない操作があっても試用者の責任ではありません。
> 気になった点はすべて「うまくいかないときの連絡方法」の欄に記録して
> ご連絡ください。
>
> ターミナル（黒い画面のアプリ）の操作は最小限になるよう手順化しています。
> 出てくるコマンドは、そのままコピーして貼り付ければ大丈夫です。

---

## 1. zipの受け取りと展開

1. 開発者からzipファイル（フォルダ一式）を受け取り、`ダウンロード`
   フォルダなどに保存します。
2. zipをダブルクリックして展開します。
3. 展開してできたフォルダを、深すぎない場所（例:
   `書類`(Documents) の直下）に移動します。
4. **重要（初回だけ）**: インターネット経由で受け取ったファイルには
   macOSが自動的に「隔離（quarantine）」の印を付け、実行時に警告を
   出すことがあります。次の手順で外します。

   1. `Launchpad` → `その他` → `ターミナル` を開く
      （または画面右上の虫眼鏡アイコンで「ターミナル」と検索）
   2. 展開したフォルダをターミナルのウィンドウに**ドラッグ＆ドロップ**
      すると、パスが自動で入力されます。それを使って次を実行します
      （`cd ` の後ろにドラッグ＆ドロップ、その後 Enter）:

      ```bash
      cd （ここにドラッグ＆ドロップしたフォルダのパス）
      xattr -rd com.apple.quarantine .
      ```

   3. エラーが出ずに次の行に戻れば完了です
      （「そのような属性はありません」等が出ても問題ありません。
      印が最初から無かっただけです）

## 2. setup_mac.sh の実行

ここから先は、上記のターミナルで同じフォルダに居るまま続けます
（違うウィンドウを開いた場合は、フォルダをもう一度ドラッグ＆ドロップして
`cd` し直してください）。

```bash
chmod +x setup_mac.sh
./setup_mac.sh
```

- 初回は Homebrew（Macによく使われる導入ツール）や Python 3.12、
  ffmpeg、各種ライブラリのインストールが走るため、**10〜20分程度**
  かかることがあります。ネット接続が必要です。
- 画面に日本語でエラーが出た場合は、その内容ごと写真かコピーで
  記録し、「6. うまくいかないときの連絡方法」に従って連絡してください。
  多くの場合、指示された1行を実行して再度 `./setup_mac.sh` を
  実行すれば解決します。
- Homebrewが入っていない場合は、案内されたコマンドで先にHomebrewを
  インストールしてから、もう一度 `./setup_mac.sh` を実行してください。
- 最後に自動で `verify_setup_mac.sh` が実行され、`[OK]`/`[NG]` の
  一覧が表示されます。**`NG` があっても、その行の対処を試してから
  続けてください**（`[--]` は「任意項目・このMacでは無くて正常」の
  意味です。例: メモリ8GB機での4Bモデル不足など）。
- 結果は同じフォルダの `setup_report_mac.txt` にも保存されます。
  フィードバックと一緒に開発者へ送ってください。

### モデルファイルの受け渡し（重要）

AIモデル本体（文字起こし・要約・チャット・話者分離用のデータファイル）は
サイズが大きいため、Gitのやり取りには含まれていません。
**開発者から別途zip等で受け取り、次の場所に配置してください:**

- `models/` フォルダ（英語ASR・言語判定・AI要約/チャット用gguf・話者分離モデル）
- `hf-cache/` フォルダ（日本語ASR = ReazonSpeech K2 用のHuggingFaceキャッシュ）

配置後にもう一度 `./verify_setup_mac.sh` を実行し、該当項目が
`[OK]` になることを確認してください。

## 3. マイク権限の許可

初めて録音を試すとき、macOSが「"ターミナル"がマイクへのアクセスを
求めています」（または Python 関連の名前）というダイアログを出します。

1. 表示されたら **「許可」** を選ぶ
2. 誤って「許可しない」を選んだ場合は、
   `システム設定` → `プライバシーとセキュリティ` → `マイク` を開き、
   ターミナル（または該当のPython）のチェックをONにして
   アプリを再起動してください
3. `./verify_setup_mac.sh` の「マイク入力」の行が `[OK]`
   （台数が1以上）になっていれば、権限は正しく通っています

---

## 4. 起動

```bash
./run_otoweave.sh
```

- ウィンドウが開けば起動成功です。デモの内容から始めたい場合は
  `./run_otoweave.sh --demo` を使ってください。
- **計測**: コマンドを実行してからウィンドウが操作できるまでの秒数を
  メモしてください（体感速度のフィードバックに使います）。

## 5. 確認シナリオ

Windows版のテスト手順（`テスト手順書.md`）と同じ観点を、Mac向けに
まとめたものです。ストップウォッチ（スマホで可）があると便利です。

### シナリオA: 録音

1. 左端の録音アイコンから「マイク」を選び、音量テストで反応を確認
2. 「録音を開始」→ 日本語で2〜3分ほど話す（教科書の音読などでOK）
3. 停止 → ノートが保存されるまでの時間を計測

### シナリオB: 文字起こし

1. 話してから文字が表示されるまでの体感の遅れ（秒）を確認
2. 表示された文字起こしを読み、10文中いくつ間違いがあるかをメモ
3. 「★」「？」「！」のマークボタンを押してみる

### シナリオC: 要約

> AI要約が使えるかどうかは自動判定されます。メモリ8GBのMacでは
> 4Bモデルは同梱されない想定のため、要約が「準備中」と案内される
> 場合があります。その場合はシナリオDへ進んでください
> （2Bでのチャットはどの構成でも使えます）。

1. 「要約を作成」を押す → 完了までの時間を計測
2. 元の話に無い内容が混ざっていないか確認
3. チャット欄で内容について1つ質問し、回答時間と妥当性をメモ

### シナリオD: 読み上げ

1. 文字起こし・要約それぞれで「🔊 読み上げ」を押す
2. `say -v Kyoko` による合成音声が再生されるかを確認
   （聞き取りやすさを5段階でメモ）
3. もう一度押して止まるか確認

### シナリオE: 話者を推定

1. 録音・保存済みのノートを開き、「話者を推定」ボタンを押す
   （録音・文字起こしが両方揃っているノートで表示されます）
2. 完了までの時間を計測
3. 結果が正しく推定されているか（実際に話した人数と一致するか）を確認
4. 必要なら「話者名を変更」で名前を分かりやすく変更する

### シナリオF: 色分け確認

1. シナリオEの後、文字起こし本文で話者ごとに行頭の名前表示の色が
   分かれているかを確認
2. 表示設定（⚙）でダーク/ライトを切り替えても見分けやすいか確認

## 6. うまくいかないときの連絡方法

- 画面が固まった・エラーが出た場合は、**そのままの画面を写真に撮って**
  ください
- 次のログファイルも一緒に共有してください（原因調査に使います）:
  - `~/Documents/LearningAccess/logs/otoweave.log`
    （起動直後のごく初期のエラーだけは一時フォルダに記録されることが
    あります。上のファイルが見つからない場合は、開発者に確認してください）
  - このフォルダの `setup_report_mac.txt`
- 連絡先: ＿＿＿＿＿＿＿＿＿＿＿＿（開発者から共有された連絡手段へ）

## 7. フィードバックの観点

Windows版と同じ `フィードバックシート.md` の欄に加えて、Mac版では
特に次の点を記録してください。

- **処理速度の体感**: 文字起こし・要約・話者推定それぞれで
  「待たされる感じがしたか」（Windows版を使ったことがあれば比較感想も）
- **発熱**: 長時間（要約や話者推定の実行中など）使ったときに
  本体が熱くなったか、ファンの音が大きくなったか
- **メモリ**: `アクティビティモニタ`（Launchpadやスポットライト検索で
  開けます）の「メモリ」タブで、OtoWeave使用中にメモリ不足の警告や
  極端なスワップが出ていないか
- **バッテリー**: 電源に繋がずに使った場合の消費の速さ（気付いた範囲で）

## 参考: Windows版との違い

- 録音・再生: sounddevice（PortAudio）を使用（Windowsは PyAudioWPatch）
- 読み上げ: `say -v Kyoko`（Windowsは Windows標準の Haruka）
- AI要約・チャット: llama-cpp-python を Metal（GPU）有効でビルド。
  `verify_setup_mac.sh` の「llama-cpp-python: Metal」の行で
  有効化されているか確認できます
- 日本語文字起こし（ReazonSpeech K2）は PyPI未公開のため、
  `setup_mac.sh` が GitHub から直接インストールします。これが失敗した
  場合は日本語のリアルタイム文字起こしのみ使えない可能性があります
  （`setup_mac.sh` 実行時の警告と、本書「6. うまくいかないときの
  連絡方法」を参照）
