# データの取り扱いと確認方法（学校・保護者のみなさまへ）

OtoWeave（オトウィーヴ）は、読み書きが苦手な生徒の学習を支えるための
授業録音・文字起こしアプリです。この文書では、

- このアプリがどんなデータを、どこに保存するのか
- なぜ「インターネットに送信しない」と言い切れるのか
- **それを信じるのではなく、ご自身で確かめる方法**

を説明します。専門用語はできるだけ避け、使う場合は説明を添えています。

---

## 1. このアプリが扱うデータと保存場所

| データ | 内容 | 保存場所 |
| --- | --- | --- |
| 録音音声 | 授業などをマイクで録音した音声 | この端末の中だけ |
| 文字起こし | 音声をアプリが文字にしたもの | この端末の中だけ |
| 要約 | 文字起こしをアプリが短くまとめたもの | この端末の中だけ |
| チャット履歴 | アプリのAIに質問した内容と回答 | この端末の中だけ |

保存場所は、標準では端末内の「ドキュメント」フォルダの中の
`OtoWeave` フォルダです（例: `C:\Users\ユーザー名\Documents\OtoWeave`）。
エクスプローラーで開けば、いつでも中身を直接確認できます。

このアプリには、会員登録・ログイン・アカウント作成といった仕組みは
ありません。データがどこかのサーバー（インターネット上のコンピューター）に
アップロードされることもありません。

## 2. 「送信しない」ではなく「送信する仕組みがそもそも無い」

### 一般的なAIアプリとのちがい

世の中の多くのAIアプリは、次のような仕組みで動いています。

```
【一般的なAIアプリ】
  あなたの端末 ──(音声や文章を送信)──▶ インターネット上のAIサーバー
  あなたの端末 ◀──(結果を受信)────── インターネット上のAIサーバー
```

音声や文章をインターネット越しに送るからこそ、「送ったデータが
どう扱われるか」が心配になります。

OtoWeaveは仕組みがちがいます。

```
【OtoWeave】
  ┌───────────── あなたの端末の中 ─────────────┐
  │  録音 ──▶ 文字起こしAI ──▶ 要約・質問AI ──▶ 保存  │
  │  （AIのプログラムとデータも、全部この中に入っている）    │
  └─────────────────────────────────────┘
                     ×── インターネットへの通信は無い
```

文字起こしや要約に使うAIモデル（AIの本体データ）を、
**アプリと一緒にまるごと端末の中に同梱**しています。だから、
外部のAIサービスに音声や文章を送る**必要そのものがありません**。
「送らないように我慢している」のではなく、「送る仕組みを最初から
作っていない」ということです。

### プログラム上の事実

- アプリのプログラムのうち、通信の命令を使っているのは1ファイル
  （`asr.py`）だけで、その通信相手はすべて `127.0.0.1` です。
  `127.0.0.1` は「この端末自身」を指す特別な番号（ループバックアドレス）で、
  この宛先への通信は**端末の外に一切出ません**。アプリ本体と、同じ端末内で
  動くAIエンジンとの「内線電話」のようなものです。
- 外部のサーバーに接続するプログラムコードは存在しません。
- 起動時に `HF_HUB_OFFLINE=1` などの設定を行い、アプリが使っている
  ライブラリ（部品プログラム）が外部に問い合わせることも遮断しています。
- セットアップも、必要なファイルをすべて同梱しているため、
  インターネット接続なしで完了します。

## 3. 自分で確かめる3つの方法

説明を信じていただくのではなく、**確認していただく**ための方法です。
どれか1つでも、3つ全部でも構いません。

### 方法(a) 機内モードデモ — ネットを切っても全機能が動くことを見る

インターネットが必要なアプリは、回線を切ると必ずどこかが止まります。
OtoWeaveは止まりません。その場で5〜10分で確認できます。

1. Windowsの画面右下のネットワークアイコンをクリックし、
   **機内モードをON**にする（またはWi-Fiをオフにする。有線LANの場合は
   ケーブルを抜く）
2. ブラウザ（EdgeやChrome）で適当なページを開き、
   **「インターネットに接続されていません」と表示されることを確認**する
   （回線が本当に切れている証拠になります）
3. OtoWeaveを起動する（切断前から起動していても構いません）
4. マイクで1〜2分**録音**し、文字起こしが出ることを確認する
5. **読み上げ**ボタンで本文が音声で読み上げられることを確認する
6. チャット欄でAIに**質問**し、回答が返ることを確認する
7. （AI要約が有効な端末なら）**要約を作成**できることを確認する
8. 確認が終わったら機内モードをOFFに戻す

インターネットが完全に切れた状態で録音・文字起こし・読み上げ・AIへの
質問がすべて動く、ということが「外部のサービスを使っていない」ことの
何よりの証拠です。

### 方法(b) 通信監視 — 通信していないことを記録として残す

配布フォルダに同梱している `verify_offline.ps1` は、OtoWeaveに関係する
プログラムの通信を数秒ごとに監視し、**端末の外への通信が0件であること**を
記録ファイルに残すスクリプトです。管理者権限は不要です。

1. `verify_offline.ps1` を右クリック →「PowerShellで実行」
   （実行できない場合は、PowerShellを開いて
   `powershell -ExecutionPolicy Bypass -File "フォルダの場所\verify_offline.ps1"`）
2. 監視が始まったら、OtoWeaveをいつもどおり使う
   （録音・文字起こし・要約・チャットなど、ひととおり）
3. 10分後（標準設定）に監視が終わり、同じフォルダの
   `offline_report.txt` に結果が追記されます
4. レポートの「外部宛て…への接続」が **0件** であることを確認する

このレポートは実施日時・端末名つきで残るので、学校への説明資料として
そのまま使えます。監視時間は `-DurationMinutes 30` のように変えられます
（`0` を指定すると Ctrl+C で止めるまで監視し続けます）。

### 方法(c) ソースコード開示 — 第三者による検証

OtoWeaveのプログラム（ソースコード）は全文開示可能です。
学校の情報担当の方や、外部の技術者など、**開発者とは関係のない第三者**が
「外部に送信するコードが無い」ことを直接確認できます。
ご希望の場合は、文末の問い合わせ先までご連絡ください。

## 4. さらに厳しくしたい場合（学校の管理者向け・任意）

「送信しない」を、Windows の設定で「**送信できない**」にすることもできます。
セットアップを管理者権限で `-BlockNetwork` オプション付きで実行すると、
Windowsファイアウォール（Windowsに標準で入っている通信の関所）に、
OtoWeave関連プログラムの**外部への送信を禁止するルール**を作成します。

```
powershell -ExecutionPolicy Bypass -File "フォルダの場所\setup_test_pc.ps1" -BlockNetwork -SkipVerify
```

- 端末内部の通信（`127.0.0.1`）はこのルールの影響を受けないため、
  アプリは全機能そのまま使えます。
- 万一プログラムに問題があったとしても、OSレベルで送信が遮断されます。
- ルール名は `OtoWeave-NoNetwork-～` で統一しており、不要になったら
  管理者権限のPowerShellで
  `Remove-NetFirewallRule -DisplayName "OtoWeave-NoNetwork-*"`
  で削除できます。

## 5. 知っておいてほしい注意点（正直に書きます）

アプリ自体は外部に送信しませんが、**端末側の環境**によっては
データがクラウドに複製されることがあります。次の点にご注意ください。

- **保存先がOneDrive等の同期フォルダになっている場合**:
  「ドキュメント」フォルダがOneDrive（Microsoftのクラウド保存サービス）と
  同期される設定の端末では、保存したデータがOneDrive経由でクラウドに
  コピーされます。これはアプリの送信ではなくWindows側の同期機能ですが、
  結果的にデータが端末の外に出ます。**アプリは起動時にこれを検知して
  警告を表示します。** 警告が出た場合は、同期の対象から
  `OtoWeave` フォルダを外すか、保存先の変更をご検討ください。
- **端末を複数人で共有する場合**:
  同じWindowsアカウントを使うと、他の人からも録音・文字起こしが
  見えてしまいます。Windowsのアカウントを人ごとに分けることを
  おすすめします。
- **削除したデータは30日間ゴミ箱に残ります**:
  ノートを削除すると、保存先フォルダ内の `_trash` フォルダに移動し、
  30日後に自動で完全削除されます。すぐに完全削除したい場合は、
  `_trash` の中のフォルダを手動で削除してください。
- **録音のマナー**: 授業などを録音する際は、事前に先生・学級の同意を
  得てください（これはデータの流出とは別の、運用上のお願いです）。

なお、アプリの動作記録（`logs\otoweave.log`）には、不具合調査用の
動作情報のみを記録し、**文字起こしの本文や発話内容は記録しない**設計です。
